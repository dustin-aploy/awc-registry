# Licensing

`awc-registry` is source-available under the Business Source License 1.1 (BUSL-1.1).

## Why This Repository Uses A More Restrictive License

`awc-registry` sits closer to submission, verification, listing, and distribution concerns than the specification and developer-tooling repositories.

That makes it closer to the commercial control layer of an AWC ecosystem:

- registry operation
- listing control
- marketplace-style distribution surfaces
- verification workflow management

For that reason, this repository does not use the same permissive licensing posture as `awc-spec` or `awc-devkit`.

## Licensing Rationale

The licensing choice is intended to:

- keep the source available for inspection and integration work
- support public understanding of the submission and verification model
- protect the registry and commercial control layer from unrestricted production reuse

See the root `LICENSE` file for the repository-specific BUSL-1.1 parameters, including the change date and change license.
